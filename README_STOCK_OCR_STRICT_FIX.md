# 股票 OCR 误识别修复

## 修复点

1. 股票 OCR 不再把券商名、按钮、逆回购、转账、行情等 UI 文案当成股票。
2. 无 6 位代码时，只从“持仓股 / 股票持仓 / 持有股票”表格区域提取候选。
3. 候选必须带有持仓数值证据，例如市值、持仓数量、价格、盈亏率。
4. 识别出名称但没有代码时，后端会尝试用股票名称搜索补代码，例如“胜利精密”自动匹配代码后再让你确认。

## 覆盖文件

- Services/StockOcrParserService.cs
- Controllers/StockController.cs

## 验证

部署后重新上传同一张中国银河证券持仓图，预览里应主要只出现“胜利精密”，不应再出现“中国银河证券 / 逆回购 / 转账 / 行情 / KB/s / WW”等候选。
