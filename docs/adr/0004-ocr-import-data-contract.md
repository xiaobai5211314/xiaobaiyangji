# OCR 导入字段契约

## 状态

Accepted

## 背景

蚂蚁基金 App 截图通过百度 OCR API 提取文本，并通过基金 OCR 导入流程（`FundController` 的 preview/confirm 流程，或后续拆分出的对应基金 OCR 服务）写入数据库。股票截图由 `StockOcrParserService` 负责解析；本 ADR 约束的是基金 OCR 导入字段契约，不绑定股票 OCR parser。需要明确哪些字段是关键契约字段，缺失时如何处理。

## 决策

OCR 导入必须提取以下字段（按优先级排列）：

| 优先级 | 字段 | 说明 | 缺失处理 |
|--------|------|------|---------|
| P0 | 当前金额 | 蚂蚁 App 显示的持有金额，可能包含买入待确认金额 | 缺失则标记截图无效 |
| P1 | 昨日收益 | 蚂蚁 App 显示的前一日收益 | 缺失则该字段为空 |
| P1 | 持有收益 | 蚂蚁 App 显示的累计收益 | 缺失则该字段为空 |
| P2 | 持有收益率 | 持有收益 / 持仓本金 | 可从持有收益和当前金额推算 |
| P2 | 确认日期 | 收益确认的日期 | 缺失则使用截图日期 |
| P3 | 待确认买入金额 | 已提交但未确认的买入 | 缺失则该字段为空 |

### 口径约束
- "当前金额"是平台持仓金额；OCR 导入时不得用其他字段逆推。
- OCR 是当前金额的校准源，不是要求用户每天导入的唯一来源；无新鲜 OCR 时，系统必须允许官方净值或盘中估值继续滚动首页当前金额。
- 存在"买入待确认金额"时，确认持仓金额必须按 `当前金额 - 买入待确认金额` 计算。
- "昨日收益"只能用于"昨日确认收益"和"盈亏日历"，不得用于反推其他金额。
- "持有收益"是蚂蚁 App 的累计收益，不得用 `当前金额 - 昨日收益` 修正。
- "买入待确认金额"未确认份额前不得参与今日收益、持有收益率本金、正式历史确认金额。

### 名称修正映射
- OCR 识别的基金名称可能与标准名称不一致（如缩写、空格差异）。
- 用户确认导入时，系统将 OCR 名称与正确基金代码/名称的映射存入 `OcrCorrection` 表。
- 后续导入同一 OCR 名称时可自动匹配已知基金代码，减少用户手动确认次数。

### 降级策略
- P0 字段缺失 → 截图标记为无效，提示用户重新截图。
- P1 字段缺失 → 降级为 `official-nav-pending` 待确认档案。
- P2/P3 字段缺失 → 留空，不影响主流程。

## 影响

- 基金 OCR 导入流程必须提取上述字段；股票 OCR 的 `StockOcrParserService` 不承担本 ADR 的基金字段契约。
- OCR 结果预览页面必须展示提取到的字段供用户确认。
- 用户确认后数据写入 `MyFundConfig` 的蚂蚁确认字段。

## 验收规则

- 提取字段与蚂蚁 App 截图一致。
- 缺失 P0 字段时明确提示"截图无效，请重新截图"。
- 缺失 P1 字段时降级为 pending 而非写入正式档案。
