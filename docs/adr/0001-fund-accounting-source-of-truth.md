# 基金正式金额事实源

## 状态

Accepted

## 背景

项目中存在多个基金金额数据来源：
- **蚂蚁 OCR 确认数据**：用户截图经百度 OCR 提取后确认的金额
- **官方净值**：基金公司 T+1 公布的正式净值
- **盘中估值**：东方财富 API 实时抓取的估算值

当多个来源同时存在时，需要明确哪个是正式事实源，避免数据冲突。

## 决策

**蚂蚁 OCR 确认数据是基金正式金额的唯一事实源。**

具体规则：
1. 当日有蚂蚁 OCR 确认数据时，所有正式金额以蚂蚁确认为准。
2. `DailyArchive` 正式档案优先使用蚂蚁确认数据写入。
3. 首页 summary 的"蚂蚁确认金额"字段直接读取蚂蚁 OCR 确认的当前金额。
4. 禁止通过 `当前金额 - 昨日收益` 或 `持有收益 - 昨日收益` 等逆推方式构造蚂蚁确认金额。

## 影响

- `DailySettlementService` 归档逻辑必须先检查当日是否有蚂蚁确认。
- `MyFundConfig` 中的蚂蚁确认字段是正式数据，非临时缓存。
- 首页 summary 和盈亏日历的金额口径必须与此一致。

## 验收规则

- 蚂蚁确认金额与蚂蚁 App 截图一致。
- 非蚂蚁确认来源不得覆盖已有的正式档案。
- 当无蚂蚁确认时，系统降级为 `official-nav-pending` 待确认状态，而非使用盘中估值。
