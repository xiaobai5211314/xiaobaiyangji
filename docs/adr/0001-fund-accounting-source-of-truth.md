# 基金金额事实源

## 状态

Accepted

## 背景

项目中存在多个基金金额数据来源：
- **蚂蚁 OCR 快照数据**：用户截图经 OCR 提取后确认的平台展示金额
- **官方净值**：基金公司 T+1 公布的正式净值
- **盘中估值**：东方财富 API 实时抓取的估算值

当多个来源同时存在时，需要同时明确"平台展示事实源"和"正式确认事实源"，避免把买入待确认金额误算进已确认持仓。

## 决策

**蚂蚁 OCR 快照是平台展示金额的最高优先校准源，不是每天唯一的当前金额来源，也不是无条件的正式确认事实源。**

正式确认口径必须先拆分：

```
平台当前金额 = 已确认持仓金额 + 买入待确认金额
```

具体规则：
1. 当日有蚂蚁 OCR 快照时，首页当前展示优先使用 OCR 快照中的平台当前金额。
2. 无新鲜 OCR 时，首页当前金额必须随官方净值或盘中估值的单基当前市值合计滚动，不能停留在旧 OCR `HoldAmount`。
3. 存在买入待确认金额时，正式确认持仓金额按 `平台当前金额 - 买入待确认金额` 计算。
4. `DailyArchive` 正式档案只能写入已确认持仓金额和已确认收益；买入待确认金额不得进入正式历史确认金额。
5. 首页 summary 的历史字段若仍命名为 `antConfirmedAmount`，语义必须是"确认持仓金额"，不是平台当前金额。
6. 禁止通过 `当前金额 - 昨日收益` 或 `持有收益 - 昨日收益` 等逆推方式构造蚂蚁确认金额。
7. 常规蚂蚁持仓截图不显示真实份额和成本价时，系统可以自动推导辅助字段：份额用官方净值从蚂蚁当前金额反推，成本金额用 `蚂蚁当前金额 - 蚂蚁持有收益` 推导。资产详情页显示 `持仓成本价` 和 `持有份额` 时，成本金额优先按 `成本价 × 份额` 校准；若平台持有收益与 `当前金额 - 成本金额` 存在差额，差额写入 `RealizedProfit`，以保持后续持有收益滚动与平台一致。派生份额和成本只能服务后续估值、收益率本金和无新鲜 OCR 时的滚动展示，不得覆盖同日 OCR 的当前金额、昨日收益或持有收益。
8. 官方净值清算后的平台展示金额按 `上一有效金额 + 已确认收益` 滚动；`份额 × 今日净值` 可用于辅助推算已确认收益，但不得直接覆盖展示金额。

## 影响

- `NavSettlementService` 官方净值确认后必须滚动更新当前持仓金额；`DailySettlementService` 归档逻辑必须先检查当日是否有蚂蚁确认。
- `MyFundConfig` 中的 OCR 字段是用户截图的结构化快照，必须保留 pending buy 拆分结果。
- 首页 summary 可以展示平台当前金额，但盈亏日历和收益率本金必须使用确认持仓金额。
- `MyFundConfig.HoldShares` 可能来自资产详情页平台字段或官方净值校准；`CostAmount` 在资产详情页有成本价/份额时按平台字段校准，在截图缺少成本价时可由蚂蚁当前金额和持有收益校准。两者都是辅助账务字段，不是高于蚂蚁 OCR 快照的事实源。

## 验收规则

- 有新鲜 OCR 时，账户总金额与蚂蚁 App 截图当前金额合计一致。
- 无新鲜 OCR 但官方净值或盘中估值已更新时，账户总金额随单基当前市值合计滚动。
- 确认持仓金额等于账户总金额扣除买入待确认金额。
- 即使份额由官方净值自动反推，有新鲜 OCR 时账户总金额仍必须与蚂蚁 App 截图当前金额合计一致，不允许被 `份额 * 净值` 的分位误差覆盖。
- 官方净值清算时，即使 `份额 * 今日净值` 与 `上一有效金额 + 已确认收益` 相差 0.01，也以后者作为展示金额和归档金额。
- 资产详情页提供成本价和份额时，成本金额应与 `成本价 × 份额` 对齐，必要时通过 `RealizedProfit` 保持持有收益与平台一致；截图未提供成本价时，成本金额应与 `当前金额 - 持有收益` 对齐。后续无新鲜 OCR 时才用该成本参与持有收益重算。
- 非确认来源不得覆盖已有的正式档案。
- 当无蚂蚁确认时，系统降级为 `official-nav-pending` 待确认状态，而非使用盘中估值。

## 外部依据

- 中国证监会深圳监管局投资者教育材料：15:00 前后的交易申请归属不同交易日，交易确认日通常为 T+1，部分特殊基金可能为 T+2/T+3。
- 天天基金帮助中心：T 日以 15:00 为界，15:00 前提交按当天收市后净值成交，通常 T+1 确认。
- 富途基金持仓资产说明：基金申购资金在交易确认前未形成持仓份额，会以买入待确认金额形式增加基金持仓金额。
