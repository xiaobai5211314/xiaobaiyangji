# 基金金额事实源

## 状态

Accepted

## 背景

项目中存在多个基金金额数据来源：
- **蚂蚁 OCR 快照数据**：用户截图经 OCR 提取后确认的平台展示金额
- **官方净值**：基金公司 T+1 公布的正式净值
- **盘中估值**：东方财富 API 实时抓取的估算值

当多个来源同时存在时，需要同时明确"平台展示事实源"和"正式确认事实源"，避免把买入待确认金额误算进已确认持仓。

## 决策

**蚂蚁 OCR 快照是平台展示事实源，不是无条件的正式确认事实源。**

正式确认口径必须先拆分：

```
平台当前金额 = 已确认持仓金额 + 买入待确认金额
```

具体规则：
1. 当日有蚂蚁 OCR 快照时，首页当前展示优先使用 OCR 快照中的平台当前金额。
2. 存在买入待确认金额时，正式确认持仓金额按 `平台当前金额 - 买入待确认金额` 计算。
3. `DailyArchive` 正式档案只能写入已确认持仓金额和已确认收益；买入待确认金额不得进入正式历史确认金额。
4. 首页 summary 的历史字段若仍命名为 `antConfirmedAmount`，语义必须是"确认持仓金额"，不是平台当前金额。
4. 禁止通过 `当前金额 - 昨日收益` 或 `持有收益 - 昨日收益` 等逆推方式构造蚂蚁确认金额。

## 影响

- `DailySettlementService` 归档逻辑必须先检查当日是否有蚂蚁确认。
- `MyFundConfig` 中的 OCR 字段是用户截图的结构化快照，必须保留 pending buy 拆分结果。
- 首页 summary 可以展示平台当前金额，但盈亏日历和收益率本金必须使用确认持仓金额。

## 验收规则

- 账户总金额与蚂蚁 App 截图当前金额合计一致。
- 确认持仓金额等于账户总金额扣除买入待确认金额。
- 非确认来源不得覆盖已有的正式档案。
- 当无蚂蚁确认时，系统降级为 `official-nav-pending` 待确认状态，而非使用盘中估值。

## 外部依据

- 中国证监会深圳监管局投资者教育材料：15:00 前后的交易申请归属不同交易日，交易确认日通常为 T+1，部分特殊基金可能为 T+2/T+3。
- 天天基金帮助中心：T 日以 15:00 为界，15:00 前提交按当天收市后净值成交，通常 T+1 确认。
- 富途基金持仓资产说明：基金申购资金在交易确认前未形成持仓份额，会以买入待确认金额形式增加基金持仓金额。
